Flow 1: matrix generation aborts on missing JSON but does not retry failed specs
If AU-4 or AU-5 Playwright runs fail mid-suite (e.g. Railway cold-start timeout), the umbrella command aborts. Playwright's own retries: 2 handles transient flake; plan relies on that. Not a gap per se, but worth knowing during /build.

i
section_catalog.py has no runtime consumer in the frontend
The catalog lives in Python (AU-2) but the TS section-selectors.ts fixture (AU-4) mirrors it by convention, not by generation. Drift risk: a new section added to parseRawData or a new drawer section won't appear in the catalog until manually added. Consider generating the fixture from catalog JSON during /build if maintenance burden grows.

Sampled
Entry: AU-1 includes this record in the stratified 15-per-uni sample.
Exit: AU-2 auditor runs.
DB-audited
Entry: db-coverage.json entry written for this uuid.
Exit: Playwright drawer + detail specs run.
Drawer PASS / Detail PASS
Entry: All expected sections visible; no CRASH.
Exit: Closed — contributes to matrix pass rate.
Drawer FAIL
Entry: title visible but ≥1 expected section missing.
Exit: AU-6/AU-8 fixes + migration-qa run --reuse-samples.
Drawer CRASH
Entry: drawer opens but tech.title never renders.
Exit: AU-6 parser fix (Array.isArray on unguarded field) + rerun.
Drawer UNREACHABLE
Entry: data-uuid selector finds no card in Discovery listing.
Exit: AU-8 adds ?openTech= deep link as alternate opener.
Detail FAIL
Entry: DetailPage renders but ≥1 expected section missing. Rare — DetailPage already works in prod.
Exit: Plan does NOT define an exit transition — DetailPage fixes are ad-hoc AU-6/AU-7 side effects.
Parity CLOSED (terminal)
Entry: Both drawer and detail PASS for a record.
Exit: none — terminal; contributes to AU-10 sign-off pass rate.
!
Dead-end state: Detail FAIL has no explicit remediation path
The plan assumes DetailPage regressions are rare (it already works) and that AU-6/AU-7 side-effects will fix any. If an audit surfaces a DetailPage-only gap, there is no AU to absorb the fix — it becomes an undeclared scope expansion during /build. Consider adding a verification step in AU-9 that flags any Detail FAIL as blocking AU-10.
